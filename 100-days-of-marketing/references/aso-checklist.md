# App Store Optimization Checklist

A focused ASO checklist for indie macOS and iOS developers. Covers what matters most — skip the enterprise analytics tooling and focus on high-impact actions.

---

## Pre-Launch ASO Checklist

### Metadata (Apple App Store)

| Element | Limit | Rules |
|---------|-------|-------|
| **Title** | 30 chars | Front-load your primary keyword. Include app name + key value word. |
| **Subtitle** | 30 chars | Secondary keyword or benefit statement. Don't repeat title words. |
| **Keyword field** | 100 chars | Comma-separated, no spaces after commas, no plurals, no duplicates from title/subtitle. |
| **Description** | 4,000 chars | First 3 lines visible without "more" — make them count. Keywords in description don't affect Apple search ranking (but affect Google Play). |
| **Promotional text** | 170 chars | Editable without app update. Use for timely messaging, sales, or new features. |
| **What's New** | 4,000 chars | Highlight user-facing changes. Don't waste on "bug fixes and improvements." |

### Metadata (macOS App Store)
Same limits as iOS. The macOS App Store uses identical metadata fields.

### Visual Assets

| Element | Guidelines |
|---------|-----------|
| **Icon** | Must be recognizable at 60x60px. Simple shape, limited colors, no text. Test against competitors on a search results page. |
| **Screenshots** | First 3 are critical — most users don't scroll. Each screenshot = one benefit, not one feature. Use captions with benefit-focused copy. |
| **Preview video** | Optional but effective. Show the app in action within the first 3 seconds. No long intros. |

## Keyword Research Process

### Step 1: Brainstorm
List 30-50 words your target users would type when looking for an app like yours. Think:
- What problem does your app solve? (e.g., "track expenses," "focus timer")
- What category is it? (e.g., "productivity app," "habit tracker")
- What do competitors call themselves?

### Step 2: Evaluate
For each keyword, assess:
- **Relevance**: Would someone searching this want YOUR app? (High / Medium / Low)
- **Competition**: How many strong apps rank for this? (check by searching the App Store)
- **Specificity**: Long-tail keywords (3-4 words) are easier to rank for

### Step 3: Prioritize
- **Primary keywords** (2-3): Put in title and subtitle
- **Secondary keywords** (5-8): Put in keyword field
- **Long-tail keywords** (5-10): Fill remaining keyword field space

### Step 4: Optimize the Keyword Field
- Use all 100 characters — don't waste space
- Separate with commas, no spaces: `keyword1,keyword2,keyword3`
- No plurals (Apple indexes both singular and plural)
- No words already in your title or subtitle (Apple combines them)
- No prepositions (a, the, and, of) unless part of a specific phrase

### Step 5: Monitor and Iterate
- Check keyword rankings after 2-4 weeks
- Replace underperforming keywords each app update
- Track which keywords drive the most impressions in App Store Connect

## Screenshot Optimization

### The Value Story (First 3 Screenshots)
1. **Screenshot 1**: Your single biggest benefit — the "aha moment"
2. **Screenshot 2**: The second most compelling feature/benefit
3. **Screenshot 3**: Social proof, breadth of features, or a differentiator

### Caption Writing
- Lead with the benefit: "Track expenses in 10 seconds" not "Expense tracking screen"
- Keep captions to 5-8 words
- Use consistent visual style across all screenshots
- Match caption tone to your tone-of-voice.md

### Design Tips
- Show real app content, not placeholder data
- Use device frames that match your target device
- Dark backgrounds often stand out in search results (test this)
- Include enough context that the screenshot tells a story alone

## A/B Testing Plan

### What to Test (in priority order)
1. **App icon** — Highest impact on conversion. Test 2-3 variations.
2. **First screenshot** — Second highest impact. Test different lead benefits.
3. **Subtitle** — Test different keyword/benefit combinations.

### How to Test
- **Apple**: Use Product Page Optimization in App Store Connect (requires iOS 15+)
- **Minimum duration**: 7 days per test
- **Minimum traffic**: ~1,000 impressions per variant for meaningful results
- **Measure**: Conversion rate (impressions to installs)

### When to Test
- Phase 1: Set your baseline (don't test yet — you need traffic first)
- Phase 3: Run your first A/B tests on icon and first screenshot
- Phase 4: Test subtitle and description changes

## Post-Launch Monitoring

### Weekly Checks
- [ ] Keyword rankings for top 5 target keywords
- [ ] Impression-to-install conversion rate (App Store Connect → Metrics)
- [ ] New reviews — read every one
- [ ] Download trend (up, flat, or down vs. previous week)

### Monthly Checks
- [ ] Keyword field refresh: swap underperformers
- [ ] Competitor check: have they changed their metadata?
- [ ] Screenshot relevance: do they still reflect the current app?
- [ ] Promotional text: update with seasonal or timely messaging

## Review Management

### Response Templates

**Positive review (4-5 stars)**:
> Thank you for the kind words, {name}! Glad {specific feature they mentioned} is working well for you. If you ever have suggestions, I'm all ears.

**Negative review (1-2 stars) — valid issue**:
> Sorry about this, {name}. {Specific acknowledgment of their issue}. I'm working on a fix — expect it in the next update. Feel free to reach out at {email} if you need help in the meantime.

**Negative review — misunderstanding**:
> Hi {name}, thanks for the feedback. It sounds like {restate their issue}. You can actually do this by {brief explanation}. If that doesn't help, email me at {email} and I'll walk you through it.

**Feature request**:
> Great suggestion! I've added this to the list. Can't promise a timeline, but I hear you. Thanks for taking the time to share this.

### Response Rules
- Respond within 24-48 hours
- Always be professional and kind, even when the review is unfair
- Never argue or get defensive
- Show you're a real person — use your name, be conversational
- Address the specific issue, don't give generic responses

## ASO Health Self-Assessment

Score yourself honestly (1-5 for each):

| Factor | Score | Notes |
|--------|-------|-------|
| Title uses primary keyword | /5 | |
| Subtitle adds value (not redundant) | /5 | |
| Keyword field uses all 100 chars | /5 | |
| First 3 screenshots tell value story | /5 | |
| Icon is recognizable at small size | /5 | |
| Description hooks in first 3 lines | /5 | |
| Reviews are responded to | /5 | |
| Metadata updated in last 90 days | /5 | |

**Score 32-40**: Strong ASO foundation. Focus on A/B testing.
**Score 24-31**: Good start. Address the lowest-scoring items first.
**Score below 24**: Prioritize ASO in Phase 1 — significant room for improvement.
